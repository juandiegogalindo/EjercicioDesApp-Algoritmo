## 5. Teoría del Juego en Base a la Programación

El ejercicio central de este repositorio se basa en el desarrollo de algoritmos, que son procedimientos finitos y comprensibles que permiten resolver problemas mediante pasos estructurados.

Desde el punto de vista programático:

1. Secuenciación de instrucciones:
Un algoritmo es una secuencia ordenada de operaciones que transforma entradas en salidas. Cada instrucción se ejecuta en un orden determinado para alcanzar un objetivo específico.
2. Estructuras de control:
- Para construir algoritmos se emplean estructuras como:
- Condicionales (if, switch), que permiten tomar decisiones.
- Bucles (for, while), que repiten acciones hasta que se cumple una condición.
- Funciones o métodos, que encapsulan pasos específicos y facilitan la reutilización.
3. Correctitud:
Un algoritmo debe producir resultados correctos para todas las entradas válidas y terminar en un número finito de pasos. La verificación de correctitud es un principio básico en diseño algorítmico.
4. Eficiencia:
Aunque ejercicios básicos no exigen optimización exhaustiva, comprender cómo medir la eficiencia computacional (por ejemplo, mediante notación Big-O) es un concepto fundamental en algoritmia académica.

Este ejercicio ayuda a desarrollar pensamiento lógico y estructurado, aplicando teoría de algoritmos a problemas concretos dentro de un programa en Java — una competencia esencial en ingeniería de software.
